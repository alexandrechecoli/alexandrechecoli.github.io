10.1 Session state

Toda vez que um aplicativo atualizado, todo o contedo das variveis criadas so zerados. Um aplicativo atualizado sempre que h uma alterao nos seus elementos: um boto clicado, um radio button selecionado, etc. Para manter variveis persistentes entre uma atualizao e outra devemos usar a Session state.

Session state pode ser pensado como um dicionrio que mantm as varivieis que no devem ser apagadas. Acessamos esse dicionrio por st.session_state. Considere o cdigo abaixo, que imprime esse dicionrio:

In[]:
import streamlit as st

st.session_state

Note que nada impresso na tela, pois o dicionrio est vazio. Podemos inserir elementos da mesma forma como fazemos com os dicionrio:

In[]:
import streamlit as st

st.session_state["v1"] = 1
st.session_state["l1"] = [1,2,3,4]
st.session_state["s1"] = "Sou uma string"
st.session_state

Dessa forma conseguimos manter a persistencia de dados na aplicao. Considere o cdigo abaixo, que mantm um contador de quantas vezes um boto foi pressionado.

In[]:
import streamlit as st


if "n" not in st.session_state: # Verificando se a chave j no est inserida
    st.session_state["n"] = 0

'''
# Aplicativo pressione o boto
'''
btn_press = st.button("Contador")
if btn_press:
    st.write("O boto j foi pressionado ", st.session_state["n"], " vezes at o momento.")
    st.session_state["n"] = st.session_state["n"] + 1

    st.session_state["dt"] = pd.read_csv(st.session_state["file"])

10.3.1 Git e github

Git um sistema de controle de verses de cdigo, usado para atualizao, sincronizao e trabalho em equipe de um mesmo arquivo. Em sua forma bsica, podemos usar o git individualmente para salvar nossos cdigos em um repositrio an web.

Cadastro no github

Um desses repsitorios o github (https://github.com/), sendo que o primeiro passo criar um usurio no site. Com um usurio criado, podemos fazer o upload de nossos cdigos para um repositorio. Para isso, precisamos tambm de um programa que faa isso instalado no nosso computador.

Git for windows

Baixar o programa git for windows em https://gitforwindows.org/.

Criando um repsitrio online

Para fazermos o upload de nossos cdigos no github, primeiro precisamos criar um repsitrio no mesmo. Ao acessar o site, clicar no boto new no canto esquerdo da tela

Em seguida precisamos fornecer um nome para o repsitorio, e clicar no boto Create repository.

Com isso o repsitorio j est criado, e um caminho de acesso fornecido. Esse caminho ser usado para fazermos o upload de nossos arquivos no repositrio.

Criando um repositrio localmente

Agora precisamos criar um repsitorio local na pasta que contm o nosso aplicativo, usando o programa que Git for windows. Para isso, procure nos aplicativos do computador Git bash. Como a primeira vez que usamos o aplicativo, necessrio configurar o usurio e e-mail (s na primeira vez de uso). Para isso digite os seguintes comandos no bash:

git config --global user.mail "<seu_@email.com>"

git config --global user.name "seu nome de usurio""

Agora, precisamos criar o repositrio localmente, na pasta em que est o nosso aplicativo. Com o prompt de comando aberto, mova-o para a pasta em que est o aplicativo com o comando:

cd "<caminho_para_a_pasta>"

Em seguida, iniciamos um repositrio git com o comando:

git init

Com o repositrio aberto adicionamos todos os documentos com os seguintes comandos:

git add .

git commit -m "alguma mensagem"

Finalmente, podemos inserir o caminho do nosso repsitrio na web, que armazenar os arquivos aps o upload, com o comando:

git remote add origin "caminho_mostrado_na_criao_do_repositorio_github"

E fazer o upload para o repsitrio web:

git push origin master

Se tudo der certo, acessando o repsitrio veremos que os arquivos j esto disponveis.

Importando o aplicativo em um servidor

Agora, com todos os arquivos no Github podemos criar uma conta no Streamlit cloud, que disponibilizar a aplicao (https://streamlit.io/cloud).

Aps criar uma conta, clique em new app.

Com isso, ser requisitado que uma integrao com a conta do Github seja feita. Logo em seguida poderemos selecionar qual repositrio e qual aplicativo queremos fazer o deploy. Se tudo der certo, o aplicativo j estar disponvel para todos, basta compartilhar o link.
