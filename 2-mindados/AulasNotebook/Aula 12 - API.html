Aula 12 - APIS¶

A sigla API deriva da expresso inglesa Application Programming Interface que, traduzida para o portugus, pode ser compreendida como uma interface de programao de aplicao. Ou seja, API um conjunto de normas que possibilita a comunicao entre plataformas atravs de uma srie de padres e protocolos.

Dessa forma, existem muitas APIs criadas para fornecer dados a outros programas. De forma geral mandamos uma requisio de alguma forma para a API e ela nos retorno algo (podemos tambm mandar requisies para realizar alteraes, etc...porm no o escopo desta aula.)

Uma requisio pode ser feita simplesmente por meio de um URL. Considere a API do OSRM (opensource routing machine), que fornece informaes geogrficas por meio de sua API. Por meio da seguinte URL:

http://router.project-osrm.org/route/v1/driving/13.388860,52.517037;13.397634,52.529407;13.428555,52.523219?overview=false

Conseguimos acessar as distncias entre as 3 localizaes (longitude,latitude) contidas na URL. A resposta vem em um formato chamado JSON (isso depende da API e dos parmetros passados). Com esse formato, conseguimos extrair os dados necessrio.

Podemos visualizar os elementos de um arquivo JSON em um visualizador online para simplificao. Considere a resposta da API acima, insira a mesma no seguinte site (http://jsonviewer.stack.hu/), e veja como a informaes est estruturada.

12.1 - Fazendo requisições de API em Python¶

Para coeltarmos a mesma informao acima direto do nosso programa em Python, precisamos usar a bibliteca requests, junto a funo get. Abaixo segue um exemplo de uso em que so coletados dados da API (https://economia.awesomeapi.com.br/last/USD-BRL,EUR-BRL,BTC-BRL), que retorna cotaes atualizadas de moedas.

In [2]:
import requests as rq

# awesome api
respostas = rq.get("https://economia.awesomeapi.com.br/last/USD-BRL,EUR-BRL,BTC-BRL")

# As respostas esto em um json que est dentro de respostas
respostas.json()['USDBRL']
