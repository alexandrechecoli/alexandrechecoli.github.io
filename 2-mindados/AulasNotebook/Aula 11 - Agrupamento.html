11.3 Definindo o nmero de grupos (clusters)

Mas como determinar o nmero de clusters? No exemplo acima definimos 3, pois j sabamos de antemo que esse era um bom nmero, dado que o conjunto foi gerado com 3 grupos. Em problemas reais, no entanto, no teremos nenhuma informaes a respeito desse nmero.

Podemos usar uma abordagem visual chamada grfico de cotovelo (elbow plot). Para isso, usamos uma medida de desempenho do agrupamento chamada WCSS(Within-Cluster Sum of Square), ou soma dos quadrados intra-grupos. Esse medida calcula a soma da distancia de todos os pontos ao centride de seus grupos. A idia que a medida que o nmero de grupos aumenta, essa distncia deve ser reduzida. Porm a partir de um determinado nmero a reduo passa a ser insignificante, de forma que esse o ponto em que aumentar o nmero de grupos no influencia mais no desempenho do agrupamento.

O atributo .inertia_ fornece o WCCS. Com isso podemos calcular os grupos para diversos valores de k, salvando os valores de WCCS em uma lista, em seguida plotamos os valores e obtemos o grfico de cotovelo.

In[7]:
l_inertia = []
for k in range(1,10):
    kmeans = km(n_clusters = k, random_state = 0).fit(X)
    l_inertia.append(kmeans.inertia_)
l_inertia

# Agora plotamos os valores de inertia e vemos como o grfico de cotovelo fica:
fig, ax = plt.subplots(1,1)
fig.set_size_inches(10,6)
ax.plot(l_inertia, marker = "o", color = "orange")
plt.show()

Pela anlise do grfico percebemos que o cotovelo est localizado no segundo grupo (como comea em zero, no terceiro), o que condiz com o esperado.

11.5.2 A influncia da escala

Sempre ao realizarmos uma anlise ade agrupamento precisamos estar atentos s escalas dos atributos. Se elas so de magnitudes muito diferentes, ento um atibuto pode dominar os outros, dado que a distncia entre os pontos realizada pela norma L2. Considere o exemplo abaixo:

In[34]:
import math as mt

idade1,renda1 = 15,10000
idade2,renda2 = 20,15000

dist = mt.sqrt((idade1 - idade2)**2 + (renda1 - renda2)**2 )
print(dist)

5000.03609986968

Alterando muito uma das variveis (idade), no afeta quase nada nas distncias, de forma que como se a varivel idade no fosse usada no agrupamento, ou ainda, estamos dando muito mais importncia varivel renda. Isso acontece pela diferena de escalas.

In[35]:
import math as mt

idade1,renda1 = 100,10000
idade2,renda2 = 1,15000

dist = mt.sqrt((idade1 - idade2)**2 + (renda1 - renda2)**2 )
print(dist)

5000.980003959224

Podemos resolver esse problema padronizando os dados das colunas. Existem diversas formas de padronizao, uma delas consiste em simplesmente dividir os dados de uma coluna pelo maior valor da mesma coluna. Dessa forma todos ficam padronizados em uma escala que varia de 0 a 1. Considere como a padronizao altera a distncia calculada no exemplo anterior.

In[38]:
import math as mt

idade1,renda1 = 15,10000
idade2,renda2 = 20,15000

maior_idade = 20
maior_renda = 10000

# Normalizando os dados
idade1_norm,renda1_norm = idade1/maior_idade, renda1/maior_renda
idade2_norm,renda2_norm = idade2/maior_idade, renda2/maior_renda

print("Dados 1 normalizados : ", idade1_norm, renda1_norm)
print("Dados 2 normalizados : ", idade2_norm, renda2_norm)

dist = mt.sqrt((idade1_norm - idade2_norm)**2 + (renda1_norm - renda2_norm)**2 )
print("Distancia normalizada : ",dist)

11.5.3 Reduo da dimensionalidade (PCA principal component analysis)

A tcnica de PCA tm por objetivo reduzir a dimensionalidade dos dados, de forma que a maior variabilidade ainda possa ser explicada. O PCA altera os atributos criando combinaes lineares dos antigos, que os representem. Ou seja, no existe uma interpretao do que os nmeros significam (antes da anlise todos os atributos poderiam ser monetrios, aps o PCA as novas variveis criadas no possuem tipo algum).
